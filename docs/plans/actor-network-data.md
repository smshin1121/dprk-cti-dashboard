## 1. Goal

Replace the reserved-slot text-only empty state on `/dashboard` (`actor-network-graph` per DESIGN.md `## Dashboard Workspace Pattern > actor-network-graph`) with a populated SNA visualization driven by a real BE endpoint, while preserving every reserved-slot discipline already locked at the design layer (no mock SVG, no fabricated data, honest empty contract when the BE returns no rows).

The endpoint surfaces co-occurrence relationships between three node kinds — actor / tool / sector — as documented in DESIGN.md, with degree-centrality node sizing and stroke-style differentiation. Filter contract matches the rest of `/api/v1/analytics/*`: `date_from`, `date_to`, `group_id[]`.

## 5. Risks & Mitigations

| Risk | Severity | Mitigation |
|:---|:---|:---|
| `compute_actor_network()` SQL is slow at production scale — actor-sector path is a 5-table join (`incident_sectors → incidents → incident_sources → reports → report_codenames → codenames`); actor-actor self-join + `COUNT(DISTINCT)` may not collapse cleanly. | High | T2 unit test runs against a fixture sized to ~1000 reports / 200 incidents; if the test takes > 500ms add `EXPLAIN ANALYZE` step. Index audit at T0: `report_codenames.report_id`, `report_techniques.report_id`, `incident_sources.report_id` (named index `ix_incident_sources_report_id` per `tables.py:319` block), `incident_sectors.incident_id`, `codenames.group_id`. If the actor-actor self-join is the slow one, consider a CTE materializing `report_id → group_id` once and self-joining the CTE. |
| FE force-layout reflows on every filter change → flicker, OR signature is stale across topology-equal-cardinality flips. | Med | L13 stable topology signature (sorted node IDs + sorted edge `source:target:weight` triples) ensures the `useEffect` re-runs iff the actual topology changes. `tick(300)` + `.stop()` produces a frozen layout. T4 has an explicit topology-change assertion to pin this contract. |
| Edge weights vary by orders of magnitude (e.g., a popular actor↔tool edge has 50× more shared reports than a sector edge), making the visualization unreadable. | Med | Edge stroke width applies `Math.log1p(weight)` scale at the FE; raw `weight` preserved on hover for analyst inspection. |
| High-weight edge silently dropped because one endpoint missed its per-kind cut by 1 rank → analyst's most-important relationship invisible. | High | L4 high-weight rescue rule + T2 explicit fixture covering the rescue path. Response carries up to 5 rescued nodes / 5 rescued edges; analyst sees the relationship even if a normal-cut endpoint missed. |
| Pact provider-state seeding for the new state collides with sibling states via unique natural keys (`groups.name`, `codenames.name`, `techniques.mitre_id`, `incident_sectors.sector_code`). | High | T8 reserves distinct names: `actor-network-fixture-G1/2/3` for groups, `actor-network-fixture-CN1/2/3` for codenames, `T9001/T9002/T9003` for technique mitre_ids (the `9xxx` range is unused in real ATT&CK), `actor-network-fixture-SEC1/2/3` for sector codes (verified format-free at T0). Pinned IDs in 999xxx range per memory `pattern_pact_literal_pinned_paths` AND distinct natural keys per memory `pitfall_pinned_id_vs_unique_name`. |
| Reserved-slot discipline silently relaxes — a future fix accidentally renders mock SVG when the BE returns empty. | High | T4 RED test pins the negative assertion (no `<svg>`/`<canvas>`/synthetic marks when `nodes.length === 0`). T5 architectural-guard test pins import contract. Failing either blocks merge. |
| SVG has 75 nodes + ~500 edges + up to 10 rescued nodes/edges → React renders >575 elements at end-of-simulation. | Low | Stopped-and-ticked simulation runs synchronously inside `useEffect`; React renders once at end-of-tick. Per-node and per-edge children memoized. If render time exceeds 100ms in T14 manual smoke, lower default `top_n_*` to 15 + cap rescue at 3 edges. |
| OpenAPI snapshot churn beyond expected scope (one path + three schemas) signals accidental drift in unrelated routes. | Low | T6 explicitly compares the diff size; anything beyond the expected delta triggers a re-read of the affected unrelated route. |
| `group_id[]` cap-breach behavior surprises analysts (default per L7 — selected actors guaranteed inclusion, can breach cap with `cap_breached: true` flag). | Med | UI legend during T10 surfaces `cap_breached` flag with a small explanation tooltip; T14 UAT captures any analyst confusion as fold candidate. |
